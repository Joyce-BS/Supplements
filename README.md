# 补剂查询网页

这是一个适合老人手机使用的静态中文网页。数据由 `supplements.json` 维护，网页可以部署到 GitHub Pages、Netlify、Cloudflare Pages 或普通静态网站服务器。

这个网页没有使用外部字体、外部图片库或外部脚本，所以在德国和中国访问时都不依赖第三方资源加载。

## 添加补剂

把真实数据写进 `supplements.json`。示例结构如下，请不要保留示例文字当作真实数据：

```json
[
  {
    "chineseName": "中文名",
    "germanName": "德文名",
    "brand": "品牌",
    "photo": "images/文件名.jpg",
    "ingredients": ["主要成分1", "主要成分2"],
    "amountPerCapsule": "每粒含量",
    "function": "功能",
    "directions": "服用方法",
    "warnings": "注意事项"
  }
]
```

网页显示用的照片建议放在 `images/optimized` 文件夹里，然后在 `photo` 字段写相对路径，例如 `images/optimized/vitamin-d.jpg`。

不要把手机原始大图直接用于网页。原图通常有几 MB，会让妈妈手机打开很慢。先压缩成宽度约 900 像素、单张 100-300 KB 左右的小图，再上传到 GitHub。

## 发给妈妈使用

不要发送 `file:///C:/.../index.html` 这种本地文件链接。这个链接只在你的电脑上有效，妈妈的手机打不开。

部署后，把网站首页的 `https://...` 链接发给她即可。她用手机浏览器打开后，可以通过浏览器菜单选择“添加到主屏幕”。如果手机浏览器支持安装提示，页面顶部也会显示“添加到主屏幕”按钮。

## 在自己电脑上预览

不要直接双击 `index.html` 预览。很多浏览器会因为安全限制拦截读取 `supplements.json`，页面就看不到补剂内容。

在 Windows 上可以双击 `start-preview.bat`，然后用它打开的 `http://127.0.0.1:4173/` 地址预览。

## 上传到 GitHub 时的图片

GitHub Pages 只需要上传网页实际使用的图片，也就是 `supplements.json` 里 `photo` 字段指向的文件。

当前应上传：

```text
images/optimized
```

原始大图可以只保存在本地备份，不建议上传到 GitHub Pages。

## 德国和中国访问

如果妈妈主要在中国使用，服务器位置和域名会影响打开速度和稳定性。

建议按下面顺序选择：

1. 国内静态托管或国内云服务器：在中国打开最稳，但通常需要实名认证；如果使用中国大陆服务器和自定义域名，可能需要备案。
2. 香港、新加坡、日本等亚洲节点：一般不需要中国大陆备案，德国和中国访问都比较折中。
3. GitHub Pages、Netlify、Cloudflare Pages：维护最简单，在德国通常没问题，但在中国访问速度和稳定性不保证。
4. 双站点方案：同一套文件同时放在一个国外站点和一个中国访问友好的站点。你维护同一个 `supplements.json`，妈妈使用国内更稳定的链接。

当前网页适合双站点部署，因为所有路径都是相对路径，复制整个文件夹到另一个服务器即可运行。

## 免费静态托管对比

前期不想增加费用时，可以先用免费静态托管试运行。

| 方案 | 中国访问稳定性 | 德国访问稳定性 | 费用 | 申请和操作难度 | 后期维护 |
| --- | --- | --- | --- | --- | --- |
| GitHub Pages | 不保证，可能慢或打不开 | 稳定 | 免费 | 中等，需要注册 GitHub、创建仓库、上传文件、开启 Pages | 适合长期记录修改历史；更新数据需要上传新版 `supplements.json` |
| Cloudflare Pages | 不保证，但有时比 GitHub Pages 稳 | 稳定 | 免费 | 中等，需要注册 Cloudflare，可连接 GitHub 自动部署，也可手动上传 | 维护较省心；后续可加自定义域名 |
| Netlify | 不保证，国内访问波动较大 | 稳定 | 免费额度通常够用 | 较低，可以拖拽文件夹上传；注册后即可试用 | 手动更新简单，但长期版本管理不如 GitHub 清楚 |

前期建议：先用 Netlify 拖拽上传或 GitHub Pages。Netlify 上手最快；GitHub Pages 更适合以后长期维护。若妈妈在中国打开不稳定，再迁移到香港、新加坡、日本或国内方案。

## 维护方式

只需要维护 `supplements.json` 和照片文件。每次修改后，刷新网页即可看到新数据；如果手机已添加到主屏幕，重新打开或下拉刷新即可更新。
